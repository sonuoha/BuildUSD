# Security Policy

## Supported Versions
- The latest stable release and the current `main` branch receive security fixes. Older versions may not be patched.

## Reporting a Vulnerability
- Please report security issues privately to security@buildusd.example (replace with the appropriate contact).
- Include: description, impact, reproduction steps, and environment details.
- Do not open public issues for security reports.

## Handling Process
- We will acknowledge receipt within 5 business days.
- We will investigate, triage severity, and work on a fix or mitigation.
- We may request additional information to reproduce or verify.

## Disclosure
- Once a fix is available, we will publish release notes and, when appropriate, CVE details.
- We aim for responsible disclosure that balances user protection and transparency.
